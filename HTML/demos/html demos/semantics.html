HTML Semantic Tags (as of HTML 5)

Semantic tags are more descriptive than vanilla 'div' tags

  • The < main > tag:
    • This tag should wrap the main content of the page - and content that is unique to that page. It shouldn't contain shared content (like nav bars etc) that exists on multiple pages.
  • The < section > tag:
    • This tag wraps/defines a specific section of the page. Thematic groupings including blog lists, contact info, and even generic content that requires its own space are considered sections (and are often differentiated with explicit 'h' tags declaring their purpose, etc.).
  • The < article > tag:
    • This tag wraps/defines any article content on the page; self-contained compositions that are indipendantly distributable, re-useable. A forum post, user comment, news or magazine article, or even an interactive widget are all things that exist outside of your webpage, and can be inserted into that page for additional, attributable content.
  • The < aside > tag:
    • This tag wraps/defines any aside content on the page. "More like this," "cited works," sections and anything else that has to do with the main content, but is tengential in nature is considered an aside.
  • The < header > tag:
    • This tag wraps/defines any header content on the page. Often, this includes the title, logo, and even navigation.
  • The < footer > tag:
    • This tag wraps/defines the footer content on a page. Often, this includes contact info, copyright notice.
